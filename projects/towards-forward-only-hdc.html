DATE 2024

This paper pushes HD encoder learning toward forward-only updates by replacing explicit feedback with a stochastic projection signal.

The project introduces SEP, an early forward-only dynamic-encoder method for hyperdimensional computing. Instead of backpropagating a detailed feature-level error, it perturbs the projection matrix with a random hypermatrix scaled by the model's classification error. The goal is simple: better HD representations at 3,000 dimensions without inheriting a backprop-style training loop.

Abstract

This is the paper where CELL starts asking whether HD encoder learning can stay adaptive without becoming backpropagation in disguise.

Classic HD learning treats the encoder as fixed infrastructure. Once the projection matrix is initialized, only the class hypervectors are retrained. That leaves a structural weakness: if the projection is poor, the model is forced to compensate by using larger dimensions or accepting a weaker representation.

Towards Forward-Only Learning for Hyperdimensional Computing introduces Stochastic Error Projection, or SEP, as a lightweight answer. SEP still trains the class hypervectors in the usual way, but it also updates the projection matrix using the average class error and a random hypermatrix instead of an explicit backpropagated correction. The method is intentionally rougher than later dynamic-encoder work, but the paper shows that this implicit signal is already enough to match strong HD baselines while staying at D = 3,000.

Motivation

Three issues make this early forward-only paper more important than its short length suggests.

The work is compact, but it is aimed at a precise gap in the HD literature: better encoder learning without importing a heavy explicit-feedback mechanism.

Static encoders lock in avoidable representation error

Once the projection matrix is frozen, retraining can only repair mistakes through the class hypervectors. The paper argues that this is too limiting for complex data and forces HD models to spend dimensions compensating for weak projections.

Explicit encoder feedback is not the only path to adaptation

Prior dynamic-encoder approaches can improve quality, but they rely on more explicit correction signals. SEP asks whether a simpler forward-only perturbation mechanism can still move the encoder toward better representations.

Dimension should not be the default repair mechanism

The paper keeps the model at D = 3,000 and frames the main win as better accuracy at lower dimensionality. That matters because the cost of encoding, similarity search, and deployment all scale with hypervector width.

Pipeline

SEP keeps the HD learning loop familiar, then inserts one stochastic step that lets the encoder move.

That added step is the heart of the paper. The model still encodes inputs, updates class hypervectors, and measures error. The difference is what happens to the projection matrix afterward.

Step 01
Encode the sample with a nonlinear projection matrix
The input feature vector is projected into hyperspace with a learned projection matrix P and a ReLU-style nonlinear encoding. This is the same structural location where static HD methods would normally stop adapting once initialization is done.
Step 02
Update class hypervectors from the softmax classification error
SEP still performs the usual class-hypervector update. The encoded hypervector is compared against the class memory, normalized with softmax, and used to produce a per-class error vector that corrects the class hypervectors first.
Step 03
Project the average error into the encoder through a random hypermatrix
Instead of explicitly tracing the error back through the features, SEP generates a random hypermatrix B with the same shape as P, scales it by the mean class error and learning rate, and binds that update into the projection matrix. Large errors induce larger stochastic movement; small errors leave the encoder more stable.
Step 04
Choose between fixed and variable stochastic guidance
The fixed-B version reuses the same random matrix across retraining, while the variable-B version regenerates it at each iteration. The paper treats these as two distinct design points: fixed B is more stable, while variable B is noisier early but more robust.
Step 05
Exploit the smaller dimension at inference time
Because SEP reaches competitive quality at D = 3,000, the paper's efficiency claim is straightforward. The learned encoder lets the model stay narrower, which directly lowers similarity-search and inference cost relative to 10,000-dimensional HD baselines.

Why the stochastic update matters

SEP is deliberately approximate. It does not tell each feature exactly how it should change. Instead, it asks whether a coarse but cheap update direction can still improve the encoder enough to move the whole model forward. The answer in this paper is yes.

Why this page should not be confused with the later forward-only paper

This DATE paper is the precursor, not the full endpoint. It introduces SEP only, establishes the core forward-only idea, and validates that implicit encoder feedback can work. The later ICCD work expands the design space from here.

Results

The evidence is compact, but it makes a clear point: implicit encoder feedback can close most of the gap without paying for a 10K model.

Because this is a short DATE paper, the evaluation is narrower than the later journal and conference extensions. The page should reflect that honestly while still surfacing the paper's main quantitative claim.

How the paper positions SEP against explicit-feedback dynamic encoder learning

Method What the paper emphasizes Main tradeoff
TrainableHD 3K Provides the closest explicit-feedback comparison at the same dimensionality. Uses a more detailed correction signal but ends the reported MNIST example below both SEP variants.
TrainableHD 10K Represents the high-dimension quality target that SEP is trying to approach. Achieves the strongest reported comparison accuracy, but pays the wider-dimension inference cost.
SEP fixed 3K Reuses one stochastic guidance matrix and converges more stably. Can be slightly less robust than regenerating the random guidance over time.
SEP var 3K Regenerates the stochastic guidance every iteration and reaches the best reported SEP accuracy. Early training is noisier and slightly slower to settle because the guidance changes each round.

The main result is qualitative as much as quantitative

The exact numbers matter, but the larger claim is architectural: encoder adaptation does not need to be feature-perfect to be useful. Even a stochastic, mean-error-driven update can meaningfully improve HD representations.

Fixed and variable B expose an important design choice

The paper does not collapse SEP into one variant. It shows that stability and robustness can be tuned by whether the random guidance matrix is persistent or regenerated, which is a useful framing for later work.

This paper's contribution is proof of viability, not a fully expanded training framework

That is exactly why it matters. It establishes that forward-only encoder learning is worth pursuing before the later work broadens the method family and pushes deeper into hardware-aware acceleration.
