Abstract

Moving HD learning past static random encoders.

Prior HDC classifiers leaned on fixed, randomly generated base hypervectors. That kept inference simple, but it also meant the encoder could not adapt to feature relationships discovered during training. The usual workaround was to increase dimensionality into the 10,000-range, which hurt efficiency and still left an accuracy gap against deep learning.

This paper introduces the first TrainableHD framework: class-hypervector errors are translated back into encoder updates, the encoder is retrained only when useful through Encoder Interval Training, and quantization-aware training keeps the learned representation deployment-friendly. The result is an HD pipeline that improves accuracy by up to 22.26 percent while preserving the lightweight inference path that makes HDC attractive for edge hardware.

Motivation

What was limiting earlier HD learners.

The paper is not just "better HDC accuracy." It attacks the core representation bottleneck: static codebooks waste dimension, ignore feature correlation, and make quantized deployment harder than it should be.

Static codebooks

Earlier encoders generated base hypervectors once and never touched them again. That random, fixed mapping assumes near-orthogonality between features even when the dataset clearly contains structure worth learning.

Dimension pressure

Because the encoder is not optimized, prior HD pipelines often need very large dimensions to stay accurate. The paper explicitly frames D = 10,000 as common practice and shows TrainableHD remains competitive even when the dimension budget drops sharply.

Edge deployment gap

HD inference is attractive on efficient hardware, but only if the representation itself can be quantized cleanly. TrainableHD treats quantization as part of the learning recipe rather than an afterthought.

Pipeline

How TrainableHD actually changes the learning loop.

The technical move is to make the encoder trainable without turning HDC into a heavy backprop-only model. The paper preserves the normal HD class-memory path, then adds a targeted way to improve the base hypervectors that define the encoder.

Step 01
Encode with trainable base hypervectors
Instead of freezing the projection matrix after random initialization, TrainableHD lets the base hypervectors evolve during training. Feature vectors are still mapped into holistic hypervectors, but the codebook is now optimized for the dataset rather than left random forever.
Step 02
Translate class error back into encoder space
The key insight is an interdimensional mapping view of HD encoding. Per-class scalar errors are turned into hypervector-form error signals, combined with binarization factors, and then used to update the base hypervectors. That is the paper's core mechanism for encoder learning.
Step 03
Run encoder updates only when they matter
Encoder Interval Training caches feature hypervectors and retrains the encoder periodically instead of every minibatch. In the paper's measurements, EIT reduces training time for base updates by 89.20 percent and repeated encoding overhead by 69.89 percent.
Step 04
Quantize the representation during training
Quantization-aware training models low-precision hypervectors, while Drift-Aware Update selectively triggers fake quantization only when weights drift enough to matter. DAU cuts quantization overhead by 84.50 percent on average with minimal accuracy change.
Results

Accuracy improved, but the stronger story is efficiency at smaller dimensions.

The paper positions TrainableHD as an accuracy-and-systems result. It closes much of the HD-versus-DNN quality gap while preserving the cheap inference path that lets HD models excel on low-power hardware.

Why this became a lineage page

This DATE paper is the clean first milestone for the later TrainableHD journal version. The core ideas are already present here: encoder learning, interval-based optimization, and quantization-aware deployment.

What stayed lightweight

The paper is careful not to improve training by sneaking in a more expensive inference pipeline. Once training finishes, TrainableHD runs with the same style of HD query-to-class similarity operations rather than adding extra neural inference stages.

Takeaways

Why this page matters in CELL's HD storyline.

  1. Point 01
    This is the early TrainableHD paper that turns encoder learning into a first-class problem. It explains where the later, fuller TrainableHD work begins.
  2. Point 02
    The page should read as both an algorithm paper and a deployment paper. Accuracy gains matter here because they are achieved without giving up HDC's efficient inference path.
  3. Point 03
    Chronologically, it is one of the clearest bridges from classic static-encoder HDC to the later CELL line on trainable, forward-only, and hardware-aware hyperdimensional learning.
