Abstract

Reducing the real bottleneck in multi-chip DNN inference.

Scaling up a single accelerator is often the wrong answer for larger neural networks. Area, power, and memory limits push practical deployments toward model partitioning across multiple devices. The problem is that once a model is split, intermediate activations become a communication burden, and those tensors can easily dominate the cost of the pipeline.

QuiltNet inserts learned encoder and decoder layers at the partition point. They act as an autoencoder that shrinks the communicated tensor, then reconstructs it on the receiving side using only layers already supported by standard accelerators. That lets the system reduce communication by orders of magnitude without demanding hardware changes, and the paper reports up to 30.5 percent higher performance and 28.4 percent better energy efficiency on multi-chip FPGA inference.

Motivation

Why partitioning alone was not enough.

QuiltNet is a systems paper about communication-first design. Its point is that model parallelism becomes much more attractive once the tensor handed between chips stops being treated as fixed and untouchable.

Single-chip scaling hits limits

Modern DNNs keep growing in layer count, tensor size, and memory pressure. Building a larger accelerator for every new network is not scalable once power, off-chip memory, and area budgets are taken seriously.

Inter-layer tensors dominate

After partitioning, each accelerator effectively becomes a producer or consumer of large intermediate tensors. Those transfers can fail to keep up with compute throughput and erase the benefit of parallel execution.

Compression usually changes hardware

A generic compression block would require hardware and software changes. QuiltNet instead makes compression part of the learned model itself, using standard fully connected or convolution layers that existing accelerators already know how to run.

Pipeline

How QuiltNet turns a split model into a communication-aware model.

The method is simple to state but carefully engineered in practice: learn a narrow boundary representation, place it where the pipeline balances best, and make the training procedure robust enough for both scratch training and transfer learning.

Step 01
Interpose data-projection layers at the split
QuiltNet inserts an encoder-decoder pair between adjacent layers. For vector-like tensors it can use a 1D fully connected autoencoder; for convolutional tensors it uses a 2D convolution autoencoder so spatial structure is preserved instead of flattened away.
Step 02
Train from scratch or retrain with freezing
If the model is already pretrained, the new autoencoder initially produces poor outputs and can trigger catastrophic forgetting. QuiltNet handles that with a two-phase schedule: first train only the autoencoder while freezing the pretrained layers, then unfreeze the full network for fine-tuning.
Step 03
Choose the partition with performance modeling
The paper uses a Paleo-style performance model to pick split locations that balance compute time and communication time across accelerators. For ResNet50, the automated decision lands between stage 3 and 4 in the reported setup.
Step 04
Pick the reduction rate from the tradeoff curve
Higher reduction shrinks communication more aggressively, but it also increases accuracy risk. QuiltNet profiles accuracy-efficiency tradeoffs directly and shows that autoencoder-only loss is a cheaper proxy for selecting a good reduction rate before full retraining.
Step 05
Deploy with no accelerator redesign
Because the learned projection layers are just standard neural layers, the split model can run on off-the-shelf accelerator stacks. The paper demonstrates the idea on FPGA-based multi-chip pipelines for ResNet50, UNet, and EfficientNet.
Results

The gains come from making communication secondary instead of dominant.

Once the transferred tensor is compressed enough, interconnect overhead stops dictating throughput. That is the key systems result across all three model families evaluated in the paper.

Representative findings from the DAC paper

Case Reported result Interpretation
Transfer learning Weight freezing improves UNet-Quilt 120x from 88.55 percent to 92.66 percent accuracy, and also nudges ResNet50-Quilt 64x from 95.22 percent to 95.62 percent. The autoencoder needs a protected warm-up phase before the rest of the pretrained model can adapt to it.
Partition search For ResNet50 with a single autoencoder at R = 64, even the worst tested placement loses only 0.47 percent accuracy; the chosen stage-3-to-4 boundary balances efficiency well. Placement matters, but the method is robust enough that the communication-aware split remains practical.
Multi-part split Using three autoencoders to split ResNet50 into four pieces still remains accurate and yields 1.44x performance improvement. The idea is not limited to a single boundary between two chips.
Training proxy Autoencoder-only training can be 76.9x faster than full-model training for ResNet50 while still serving as a useful proxy for reduction-rate selection. The workflow is realistic enough to use when exploring deployment options.

No hardware trickery required

The paper deliberately avoids specialized codec hardware. Its compression block is a learned model component, so standard accelerator kernels are enough to support the method end to end.

An early systems-thread anchor

Chronologically, QuiltNet is one of the clearest early pages for CELL's systems work. It shows the lab treating data movement as the first-order design target rather than an afterthought behind raw compute throughput.

Takeaways

Why this page matters in CELL's systems thread.

  1. Point 01
    QuiltNet reframes model partitioning as a learned communication problem. The project is about making the split boundary itself trainable and deployment-aware.
  2. Point 02
    For the website, this should read as a multi-chip systems page, not just an autoencoder page. The important shift is that tensor movement, not only compute, sets the throughput ceiling.
  3. Point 03
    It also broadens the project list beyond HD computing by showing the same CELL efficiency mindset applied to accelerator partitioning, model deployment, and hardware-conscious training workflows.
