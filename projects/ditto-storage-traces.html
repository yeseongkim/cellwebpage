Abstract

Trace generation is treated as a controllable generative modeling problem.

DiTTO, short for Diffusion-based Trace generation and Temporal Outpainting, targets a very practical systems problem: realistic storage traces are expensive to collect, often private, and difficult to cover across the range of configurations needed for evaluation. Existing synthetic tools are usually template-driven, while earlier learning-based approaches still depend heavily on hand-crafted features.

The paper replaces that workflow with a diffusion-based generator that operates on structured multi-device trace representations. Raw access logs are converted into multi-channel images, numeric workload targets are embedded through a dedicated conditioning model called CHIP, and outpainting extends traces beyond the original time horizon. The result is a system that aims to be realistic, diverse, and explicitly steerable by measurable properties such as read/write ratios, request rates, device utilization, burstiness, and temporal dependencies.

Motivation

The storage-systems problem is not just realism. It is realism plus controllability.

A synthetic workload generator is only useful if researchers can ask for very specific operating conditions and still get traces that look like real systems behavior. DiTTO is built around that double requirement.

Real trace collection is expensive and constrained

Distributed storage workloads are valuable for analyzing caching, load balancing, and system bottlenecks, but direct collection carries instrumentation cost and privacy risk. That means the hardest traces to get are often the ones needed to stress new optimizations.

Template-based synthesis is easy to run but hard to steer precisely

Tools such as benchmark-style synthetic generators can replay predefined patterns, but they struggle to model evolving cross-device behavior. Machine learning alternatives improve realism, yet often still depend on expert-crafted features instead of letting the user specify measurable workload targets directly.

Pipeline

DiTTO is a four-part pipeline: represent, condition, denoise, then extend.

The most interesting part of the paper is that it does not condition a diffusion model with vague text prompts. It builds an explicit numeric control path for workload properties and couples that with a representation designed for sparse, multi-device traces.

Step 01
Turn logs into structured images
Per-device access logs with timestamps, operation types, and device identifiers are segmented over time. The x-axis is time, the y-axis is the device dimension, and separate channels encode read versus write behavior.
Step 02
Reduce sparsity without destroying structure
Storage workloads are extremely sparse, so the paper adds local Gaussian noise around access events. This is not cosmetic; it gives the diffusion model enough local signal to learn temporal and cross-device context instead of mostly seeing empty space.
Step 03
Condition generation with CHIP
CHIP, short for Contrastive Hyperconfiguration-Image Pretraining, maps numeric workload configurations into a latent space aligned with the image-form trace representation. This gives DiTTO explicit control over quantifiable properties rather than relying on loosely interpreted prompts.
Step 04
Outpaint for arbitrary trace length
Instead of generating a fixed window once, DiTTO uses outpainting to continue the trace conditioned on the boundary state of the earlier segment. That allows long traces while preserving continuity in burst behavior, device coordination, and workload fluctuations.

Why CHIP matters

The conditioning path is the differentiator. The paper argues that storage-trace generation needs direct control over numeric targets such as read/write ratios and dominant devices. CHIP uses contrastive learning so similar configurations sit near similar trace representations, which makes the diffusion model easier to guide with precise workload constraints.

Why outpainting matters

Fixed-length samples are not enough for systems studies. Outpainting lets DiTTO extend traces while preserving what came before, which is essential if the generated output is going to be used for longer-running evaluations instead of just short qualitative demonstrations.

What the paper validates

The main empirical claim is that DiTTO can stay close to user-specified workload characteristics while remaining realistic and diverse. The t-SNE analysis reported in the paper places generated traces near the original trace distribution instead of collapsing into a single synthetic mode.

What the page should emphasize

This is a systems-meets-generative-AI paper, not a generic diffusion showcase. The value is that synthetic traces become a controllable systems artifact: something storage researchers can ask for, vary systematically, and extend over time without collecting new production logs for every scenario.

Takeaways

Why DiTTO fits the broader CELL project portfolio.

  1. Point 01
    It uses generative AI for a systems bottleneck rather than for user-facing media. The synthetic artifact here is a workload trace that can drive downstream design and optimization studies.
  2. Point 02
    The conditioning mechanism is the core technical move. By pushing measurable workload properties into the latent guidance path, DiTTO treats controllability as a first-class requirement instead of a post hoc evaluation metric.
  3. Point 03
    For CELL's website, this project should read as a bridge between generative modeling and infrastructure research: diffusion models are being used to create better virtual playgrounds for storage-system evaluation.
