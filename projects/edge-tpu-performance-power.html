Abstract

Why Edge TPU prediction needed more than MAC counts.

This paper starts from a practical problem: when deploying deep learning on resource-limited edge hardware, the usual rule of thumb is to estimate performance from the number of MAC or FLOP operations. The authors show that this is not reliable for Google Edge TPU because actual behavior depends strongly on layer type, memory placement, and compiler decisions about on-chip versus off-chip usage.

To make this concrete, the paper characterizes 10,158 models and then builds PETET, a machine-learning-based estimator that predicts execution time, power, and energy from model descriptions. PETET learns memory behavior with random forests, then uses those estimates to predict the final metrics online. The resulting framework stays within 10 percent error for both fully connected and convolutional networks and reaches 6.51 percent average energy error on practical MobileNet and VGG models.

Motivation

What simple performance models were missing.

The paper is valuable because it treats Edge TPU as a real closed-source deployment target. It does not assume the compiler, memory hierarchy, and power behavior can be collapsed into one linear compute-cost formula.

MAC count is not enough

For both fully connected and convolutional models, the paper finds that inference time does not scale linearly with the number of MAC operations. The observed curves are stepped, clustered, and highly dependent on model structure.

Memory placement changes everything

Edge TPU uses both on-chip SRAM and host-side off-chip memory. When the compiler spills tensors or parameters off chip, communication overhead changes the latency curve substantially.

Design exploration needs fast estimates

Compiling every candidate model and measuring it physically is too slow for hyperparameter search or deployment planning. The paper therefore aims for online prediction directly from model descriptions.

Pipeline

How PETET predicts performance, power, and energy.

PETET stands for Performance/power Estimation Technique for Edge TPU. Its structure reflects the paper's main claim: memory behavior has to be estimated first, and only then can execution time and power be predicted accurately.

Step 01
Characterize the hardware broadly
The paper generates and measures more than ten thousand TensorFlow models, covering MAC counts from 192 to 4.7 x 10^11. It records inference time and power on Edge TPU for both fully connected and convolutional settings.
Step 02
Extract model-level features
For each candidate model, PETET extracts network type, number of layers, input size, parameter count, and MAC count. These are the descriptors used to infer memory use and final runtime behavior.
Step 03
Predict on-chip and off-chip memory use
Random forest estimators model the hidden memory-allocation behavior of the Edge TPU toolchain. PETET supports both compile-based mode, which reads memory from actual compilation, and prediction-based mode, which estimates it directly without compiling.
Step 04
Predict time and power from memory plus compute
Once memory usage and MAC count are available, PETET predicts execution time and power with separate learned models. Energy then follows from those two outputs, enabling one-shot hardware-aware design estimates.
Step 05
Validate on real architectures, not just synthetic ones
The framework is finally cross-validated on practical MobileNet and VGG families at multiple input sizes and width multipliers, showing that the learned estimator generalizes beyond the synthetic characterization set.
Results

The core result is explanatory accuracy, not just a low average error number.

PETET matters because it captures the actual sources of Edge TPU variability. The reported errors are low, but the deeper contribution is identifying which factors create those non-linear runtime and power patterns in the first place.

Representative findings from the DATE 2022 paper

Observation Reported evidence Implication
FC models are often memory-bounded For the studied FC networks, performance shifts strongly once off-chip memory is used, and the paper notes a clear threshold behavior around image width W = 48 in one setup. Estimators need memory-placement awareness, not just arithmetic intensity.
CONV behavior depends on multiple interacting factors Convolutional inference is influenced by MAC count, on-chip memory usage, and off-chip memory usage, with no single factor always dominating. A single scalar complexity metric is too weak for accurate prediction.
Prediction-based mode is practical PETET's PB mode avoids repeated compilation while still predicting time, power, and energy with low error, and differs negligibly from compile-based quality by about 3.94 percent. This makes the model suitable for online search and deployment tooling.
Real-model validation holds up Across MobileNet V1/V2 and VGG16/VGG19 configurations, the average energy-estimation error is 6.51 percent. The learned model generalizes beyond the synthetic characterization set.

A systems-modeling page, not just a benchmark page

The paper's strength is that it explains why Edge TPU behaves the way it does. The step-wise latency, the off-chip threshold effects, and the converging power curves are what make PETET defensible as more than a curve-fit.

Useful for real search loops

The prediction-based mode is what makes this project page important for the website. It turns characterization into a reusable tool for hardware-aware architecture and model-search decisions without requiring full compiler runs each time.

Takeaways

Why this page matters in CELL's systems thread.

  1. Point 01
    This page shows CELL working at the measurement-modeling boundary of systems research. The contribution is not a new network architecture, but a way to reason correctly about a commercial edge accelerator.
  2. Point 02
    For the website, PETET should be framed as a deployment-enabling model. The key move is learning hidden memory behavior well enough to make online power and latency estimates credible.
  3. Point 03
    Chronologically, it also complements the neighboring Edge TPU HDC page: one paper adapts a learning algorithm to the device, while this one builds the predictive systems understanding needed to use the device intelligently.
