CascadeHD: Efficient Many-Class Learning Framework Using Hyperdimensional Computing

CascadeHD revisits HDC for many-class problems by fixing how class hypervectors are trained and by organizing inference as a tree that spends more work only on confusing classes.

Abstract

Scaling HDC when the number of classes gets large.

Hyperdimensional computing is attractive on edge devices because training and inference are lightweight, parallel, and hardware-friendly. The problem is that most prior HDC work focused on small-class settings. Once the label space grows, standard HDC retraining struggles to separate confusing classes and inference becomes expensive because every query must be compared against every class hypervector.

CascadeHD addresses those two issues with two orthogonal ideas. MAny class Similarity Scaling, or MASS, replaces the traditional two-class correction rule with an all-class update based on similarity gaps, which stabilizes many-class learning. Waterfall training then converts the flat class list into a hierarchical inference structure that merges similar classes into latent decision nodes and routes difficult cases more selectively. Together they improve accuracy by up to 18 percent and improve runtime by 32 percent over prior HDC, while still delivering very large efficiency gains over DNN baselines on Jetson devices.

Motivation

Why many-class HDC needs different training and different inference.

CascadeHD is important because it identifies two separate failure modes in prior HDC systems and fixes both: model quality falls when classes are numerous and decision boundaries get crowded, and inference cost rises because every class must still be checked exhaustively.

Coarse retraining breaks down

The standard HDC retraining rule only updates two class hypervectors for a misclassified sample. In many-class settings that is too weak, often too slow to converge, and can leave narrow or unstable margins between classes.

Similarity search scales poorly

As K grows, HDC inference must compute distances to every class hypervector. The result is that one of HDC's main benefits, cheap inference, erodes exactly when the classification problem becomes more realistic.

Confusing classes need extra structure

Not every class pair is equally hard. CascadeHD uses that fact to merge similar classes into latent categories and reserve fine-grained comparisons for the subsets of classes that are actually hard to tell apart.

Pipeline

How CascadeHD changes both training and inference.

The paper keeps the familiar HDC flow of encoding into a high-dimensional hyperspace, but then replaces the old retraining rule and reorganizes the final classifier into a learned cascade.

Step 01
Encode with a strong nonlinear HDC map
Samples are encoded into D = 10,000 dimensional hypervectors using a nonlinear encoding built from random base hypervectors, element-wise accumulation, cosine nonlinearity, and binarization. The goal is to expose latent interactions between features before class learning starts.
Step 02
Train all classes with MASS
Instead of only correcting the true class and the current winner, MASS computes similarities to all class hypervectors, builds a target one-hot similarity vector, and updates every class proportionally to its distance from that target. This produces faster convergence and stronger separation in many-class settings.
Step 03
Learn a waterfall hierarchy from class confusion
After training the flat class model, CascadeHD uses a validation set and class-hypervector similarity analysis to identify confusing class pairs. Similar classes are merged into latent categories, producing a tree of decision nodes instead of a single flat K-way comparison.
Step 04
Retrain in-node classifiers and add proxy nodes
Each merged node gets its own in-node binary classifier. If upper-level routing errors become important, proxy decision nodes can be added so selected nodes classify more than two classes and give the model a second chance before a mistake becomes unrecoverable.
Results

CascadeHD closes most of the accuracy gap to DNN while preserving HDC's edge efficiency.

The evaluation spans PLANT, EXTRA, ISOLET, MNISTMIX, and AUDIO on Jetson TX2 and Xavier. The paper separates the benefit of MASS from the benefit of waterfall modeling, which makes the result much clearer than a single aggregate number.

Representative findings from the DAC 2021 paper

Theme Reported result Interpretation
MASS accuracy CascadeHD with MASS achieves accuracy comparable to AutoKeras-searched DNNs while significantly outperforming the baseline HDC retraining across PLANT, EXTRA, ISOLET, MNISTMIX, and AUDIO. The main improvement comes from treating many-class retraining as an all-class similarity-scaling problem rather than a two-class mistake correction.
HDC versus DNN efficiency On Jetson TX2 and Xavier, CascadeHD achieves 61.4x and 65.6x speedup over DNN inference and 69.7x and 76.0x energy-efficiency improvement. The paper preserves HDC's edge-deployment advantage even while moving much closer to DNN accuracy.
Waterfall benefit The hierarchical model reduces runtime by 22.4 percent on average and saves 21.3 percent energy compared with the baseline flat HDC inference. The tree structure makes inference cheaper by avoiding exhaustive all-class similarity checks at every query.
Proxy-node recovery For AUDIO, setting P = 10 proxy nodes recovers 1.87 percent accuracy while still keeping a 48 percent efficiency improvement over baseline HDC. The hierarchy can be made more accurate again at a limited performance cost when root-level mistakes are especially expensive.

An HDC scaling page

CascadeHD is one of the clearest pages on the site for how CELL turned HDC from a small-class, toy-friendly method into something more robust for realistic label spaces.

Why both pieces matter

MASS and waterfall training solve different problems. MASS fixes representation quality and training convergence; the waterfall model fixes inference cost and, in some cases, can even improve accuracy by focusing on confusing class subsets.

Takeaways

Why this page matters in CELL's HD line.

  1. Point 01
    This is an early anchor for CELL's work on making HDC accurate enough for harder real-world classification problems without abandoning efficiency.
  2. Point 02
    For the website, CascadeHD should read as a dual contribution: a better many-class retraining rule and a better inference structure. Those are separate design moves with separate payoffs.
  3. Point 03
    Chronologically, this page also foreshadows later CELL work on trainable, hierarchical, and deployment-aware HDC systems. It is one of the first pages where HDC is clearly treated as a systems problem as much as a representation problem.
